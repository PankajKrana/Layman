# Layman

Business, tech, and startup news made simple.

## Tech Stack
- iOS: SwiftUI + MVVM
- Backend/Auth/Storage: Supabase
- News: NewsData.io free tier
- AI: Gemini / OpenAI / Groq via runtime provider fallback

## Features
- Welcome screen with swipe-to-start
- Email/password authentication with Supabase
- Session persistence across launches
- Home feed with featured carousel + today picks
- Real search on Home and Saved screens
- Article detail screen with:
  - 2-line headline
  - full-width image
  - 3 swipeable layman cards
  - dots + page indicator
  - in-app original article modal (SFSafariViewController)
- Ask Layman chat:
  - initial bot message
  - 3 contextual suggestion chips
  - real AI responses (1-2 sentence style)
- Saved articles synced with Supabase
- Profile with sign out and delete account action

## Setup
1. Clone the repository.
2. Copy config template:
   - `Layman/Layman/Config/Secrets.xcconfig.example` -> `Layman/Layman/Config/Secrets.xcconfig`
3. Fill required keys in `Secrets.xcconfig`:
   - `SUPABASE_URL`
   - `SUPABASE_ANON_KEY`
   - `NEWSDATA_API_KEY` (or `NEWS_API_KEY`)
   - AI keys: one or more of `GEMINI_API_KEY`, `OPENAI_API_KEY`, `GROQ_API_KEY`
4. Open the Xcode project and run on iOS simulator/device.

## Supabase Requirements
- Enable Email/Password auth.
- Create bookmarks table (example):

```sql
create table if not exists bookmarks (
  id bigint generated by default as identity primary key,
  user_id uuid not null,
  article_title text not null,
  article_url text not null,
  article_image_url text,
  created_at timestamp with time zone default now()
);
```

- Enable RLS and apply policies so users can read/write only their own rows:

```sql
alter table bookmarks enable row level security;

create policy "select own bookmarks"
on bookmarks for select
using (auth.uid() = user_id);

create policy "insert own bookmarks"
on bookmarks for insert
with check (auth.uid() = user_id);

create policy "delete own bookmarks"
on bookmarks for delete
using (auth.uid() = user_id);
```

## AI Provider Notes
- Set `AI_PROVIDER` to `gemini`, `openai`, or `groq`.
- App automatically falls back across configured providers if one fails.
- Gemini model auto-resolution is included to avoid stale model-name failures.

## Security Notes
- Do not commit `Secrets.xcconfig`.
- Keep API keys only in local config or CI secrets.
